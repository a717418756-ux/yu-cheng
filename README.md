## 專案簡介

本平台是一個純前端的漸進式網頁應用程式（PWA），部署於 GitHub Pages，可離線使用、可安裝到手機桌面。所有資料儲存在使用者裝置本機（IndexedDB），不上傳伺服器，兼顧隱私與離線可用性。

平台針對長時間備考的使用情境設計，特別支援電子紙裝置（如 Boox 閱讀器）的高對比無動畫模式，減少螢幕殘影與延遲。

### 三大分區

| 分區 | 內容 |
|------|------|
| **考試區** | 題庫練習、間隔複習、法規資料庫 |
| **休閒區** | 電子書閱讀器、影音庫 |
| **成長區** | 英語學習庫、運動健康庫 |

## 主要功能

### 📝 題庫與練習
- 多科目題庫管理，支援單選、多選、題組題、申論題
- 四層導覽：年度 → 考科 → 考試別 → 題目（考試別含警佐班／升官等／警大二技／三等考試等，未建題會標示「尚未新增」）
- 間隔複習系統（遺忘曲線），依危險度排程
- 模擬考、快刷模式
- 大量匯入（智慧解析選項符號，含 PDF 特殊字元、PUA 私用區字元）
- 題幹標記語法（填空、畫線、粗體，見下方說明）
- 重複題防護、依條件批量勾選刪除、全文搜尋（含搜尋語法說明）

### 🗃️ 法規資料庫
- 法規條文 / SOP / 補充資料 / 函釋分類
- 編 → 章 → 節 三層階層分組顯示
- 條號自動排序（支援「第N條之M」）、相關法條互連

### 📚 電子書閱讀器
- 支援 epub、PDF 格式
- 閱讀進度記憶（顯示「章節名 · 百分比」）
- 書架式陳列，書本依實際比例顯示
- 電子紙專屬閱讀主題

### 🎬 影音庫
- 影片、音頻課程、錄音管理
- 黑膠唱片風格音頻播放器、迷你播放列
- 縮圖按需載入、清單分頁、音訊壓縮選項

### 🗣️ 英語學習庫
- 文字／PDF／OCR 三種上傳方式
- TTS 逐句高亮朗讀（整合 Azure 語音）
- 發音評估與跟讀練習，記錄練習歷史與趨勢

### 🏃 運動健康庫
- 運動時長、消耗／攝取熱量、步數記錄
- 熱量結餘計算、雙擊卡片快速編輯
- 預留 Health Connect（三星健康）原生接入

### 🏠 首頁總覽
- **成長軌跡熱力圖**：過去 35 天，以「學習時間 + 每日任務完成度」綜合評分（差→優）
- **學習規劃卡**：里程碑（分階段複習計畫）與計畫表（行事曆時間軸）tab 切換
- **每日任務**：圓形進度環、可綁定自動完成條件（按重新整理鈕比對資料自動勾選，防偷懶）
- **成長詳情**：點熱力圖某天，顯示學習圓餅 + 運動數據 + 每日任務達成

### 📊 統計與分析
- **高頻關鍵概念**：統計答錯題目的關鍵概念（keywords），依錯誤次數排序，字體大小反映熱度
- AI 分析匯出（含記憶斷層偵測：同關鍵字短期內反覆答錯）

### ⚙️ 系統功能
- 雲端備份（Google Apps Script）、倒數計時器、使用時間統計
- 深色模式 / 電子紙模式雙主題
- 骨架屏載入、觸覺回饋

## 技術架構

| 項目 | 技術 |
|------|------|
| 前端 | 原生 HTML / CSS / JavaScript（無框架） |
| 資料庫 | IndexedDB（透過 Dexie.js） |
| 離線 | Service Worker（PWA） |
| 電子書 | epub.js |
| 壓縮 | JSZip、Web Audio API |
| 語音 | Azure Cognitive Services（經 GAS 代理） |
| 雲端備份 | Google Apps Script + Google Sheets |
| 部署 | GitHub Pages |

### 設計原則
- **離線優先**：所有資料存本機，無網路也能完整使用
- **記憶體優化**：大型資源按需載入、用完即釋放
- **電子紙友善**：eink 模式停用所有動畫與過場

### 程式碼標記
主要 JS 模組採用區段標題標記，搜尋 `【` 可列出所有功能區段。

### data.js 的特殊結構（重要）
`data.js` 以 **IIFE 包裝**（`(function(){...})()`），內部函式預設不是全域。要給 HTML 的 `onclick` 使用的函式，**必須列入檔尾的 `DataMod` 白名單**，才會掛上 window 變全域。新增此類函式時若漏列白名單，會出現 `xxx is not defined` 錯誤。

## 題目輸入語法

### 標記語法（題幹與選項通用）
新增題目時可在題幹或選項使用以下標記，答題時自動呈現樣式：

| 標記 | 效果 |
|------|------|
| `___`（3個以上底線） | 填空底線 |
| `[[提示]]` | 填空（含淡色提示字） |
| `**文字**` | 畫線強調 |
| `//文字//` | 粗體 |

範例：`國父姓 **孫**，______ 是他的字，//三民主義// 為其思想。`

（新增題目與大量匯入頁的標題旁，皆有 ⓘ 說明按鈕可隨時查看。）

### 選項符號自動辨識
大量匯入時，下列選項標記都會自動解析為選項 A/B/C/D：
- 半形／全形括號字母：`(A)`、`（A）`
- 圈圈字母／數字：`Ⓐ`、`①`、`⒜`
- 私用區（PUA）特殊字型字元：`U+E18C`~`U+E18F`（特定題庫字型用）
- 行首重複的未知符號（自動偵測為選項分隔）

### 關鍵概念（keywords）
新增題目時的「關鍵概念」欄位，存為 `keywords`，是**高頻關鍵概念分析**的資料來源。建議每題填寫，分析才精準。

## 開發規範

### 版本號（最重要）
更新時必須**同時**修改兩處 `APP_VERSION`，且保持一致：
- `js/db.js`（約第 327 行）
- `sw.js`（約第 10 行）

### 檔案分工
| 類型 | 放哪裡 |
|------|--------|
| 固定的頁面骨架 | HTML（index.html） |
| 資料驅動、會變動的內容 | JS（動態生成） |
| 所有樣式、位置、排版 | CSS |

> 一次功能改動常需同時修改 2–3 個檔案，部署時務必全部上傳，漏傳任一檔功能即不完整。

### 品質要求
- **無死碼**：移除不再使用的函式與樣式
- **無重複定義**、**不誤傷**、**先確認根因再修**
- 每次改動後驗證：
  - JS 語法（`node --check`）
  - CSS 括號配對與孤立選擇器掃描
  - **data.js 新增 onclick 函式須確認已列入 DataMod 白名單**（防 not defined）
  - 版本號一致性

## 版本與更新機制

採用 Service Worker 快取，版本以 `APP_VERSION` 控管。新版部署後，Service Worker 偵測到版本變化並提示更新。

### 關於「清快取」
瀏覽器有兩層快取：
1. **Service Worker 快取**：由本專案的更新機制管理
2. **瀏覽器 HTTP 快取**：瀏覽器自行管理，程式碼無法控制

更新後若畫面未變，通常是第 2 層 HTTP 快取仍提供舊檔。此時需**完全關閉應用程式後重開**，或手動清除網站資料。這與更新功能是否正常無關，屬瀏覽器層級的快取行為。

## 未來規劃

### Capacitor 原生封裝
計畫以 Capacitor 將 PWA 封裝為 Android App，以接入原生能力：
- **Health Connect 整合**：自動讀取三星健康的運動數據（純 PWA 無法讀取，需原生 app 作橋樑）
- **直接開啟外部 App**
- **OTA 線上更新**：網頁內容更新免重新安裝

| 改動類型 | 是否需重新安裝 |
|----------|----------------|
| 網頁內容（HTML/CSS/JS） | 否（可用 OTA） |
| 新增原生外掛 | 是 |
| 改 App 圖示／名稱／權限 | 是 |
| 升級 Capacitor 核心 | 是 |

### 其他規劃
- 題目自動抽取關鍵概念（免手動標 keywords）
- 英語單字本間隔複習
