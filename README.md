## 專案簡介

本平台是一個純前端的漸進式網頁應用程式（PWA），部署於 GitHub Pages，可離線使用、可安裝到手機桌面。所有資料儲存在使用者裝置本機（IndexedDB），不上傳伺服器，兼顧隱私與離線可用性。

平台針對長時間備考的使用情境設計，特別支援電子紙裝置（如 Boox 閱讀器）的高對比無動畫模式，減少螢幕殘影與延遲。

### 三大分區

| 分區 | 內容 |
|------|------|
| **考試區** | 題庫練習、間隔複習、法規資料庫 |
| **休閒區** | 電子書閱讀器、影音庫 |
| **成長區** | 英語學習庫、運動健康庫 |

## 主要功能

### 📝 題庫與練習
- 多科目題庫管理，支援單選、多選、題組題
- 間隔複習系統（遺忘曲線，7 級間隔），依危險度安排複習
- 三層導覽（年度 → 科目 → 題目）、模擬考、快刷模式
- 重複題防護、依條件批量勾選刪除、無限捲動載入
- 全文搜尋索引

### 🗃️ 法規資料庫
- 法規條文 / SOP / 補充資料 / 函釋分類
- 編 → 章 → 節 三層階層分組顯示
- 法條檢視與編輯切換、相關法條互連
- 條號自動排序（支援「第N條之M」）、重建條號索引

### 📚 電子書閱讀器
- 支援 epub、PDF 格式
- 閱讀進度記憶（顯示「章節名 · 百分比」）
- 書架式陳列、最近閱讀
- 電子紙專屬閱讀主題

### 🎬 影音庫
- 影片、音頻課程、錄音管理
- 黑膠唱片風格音頻播放器、迷你播放列
- 縮圖按需載入（節省記憶體）
- 上傳時可選音訊壓縮、清單分頁

### 🗣️ 英語學習庫
- 文字／PDF／OCR 三種上傳方式
- 自動切分句子，TTS 逐句高亮朗讀（整合 Azure 語音）
- 發音評估與跟讀練習，記錄練習歷史與趨勢

### 🏃 運動健康庫
- 運動時長、消耗／攝取熱量、步數記錄
- 熱量結餘計算
- 雙擊卡片快速編輯數值
- 預留 Health Connect（三星健康）原生接入

### 🏠 首頁總覽
- 成長軌跡熱力圖（過去 35 天，差→優漸層）
- 每日任務（圓形進度環、可編輯、達成歷史查閱）
- 成長詳情整合學習時間與運動數據

### ⚙️ 系統功能
- 雲端備份（透過 Google Apps Script）
- 倒數計時器、使用時間統計
- 深色模式 / 電子紙模式雙主題
- 骨架屏載入、觸覺回饋

## 技術架構

| 項目 | 技術 |
|------|------|
| 前端 | 原生 HTML / CSS / JavaScript（無框架） |
| 資料庫 | IndexedDB（透過 Dexie.js） |
| 離線 | Service Worker（PWA） |
| 電子書 | epub.js |
| 壓縮 | JSZip、Web Audio API |
| 語音 | Azure Cognitive Services（經 GAS 代理） |
| 雲端備份 | Google Apps Script + Google Sheets |
| 部署 | GitHub Pages |

### 設計原則

- **離線優先**：所有資料存本機，無網路也能完整使用
- **記憶體優化**：大型資源（音訊、縮圖）按需載入、用完即釋放
- **電子紙友善**：eink 模式停用所有動畫與過場，消除殘影

## 開發規範

以下是維護本專案的核心規範：

### 版本號（最重要）
更新時必須**同時**修改兩處 `APP_VERSION`，且保持一致：
- `js/db.js`（約第 327 行）
- `sw.js`（約第 10 行）

版本號變動後，Service Worker 才會偵測到新版並提示更新。

### 檔案分工
| 類型 | 放哪裡 |
|------|--------|
| 固定不變的頁面骨架 | HTML（index.html） |
| 資料驅動、會變動的內容 | JS（動態生成） |
| 所有樣式、位置、排版 | CSS |

> 因此一次功能改動常需同時修改 2–3 個檔案，部署時務必全部上傳，漏傳任一檔功能即不完整。

### 品質要求
- **無死碼**：移除不再使用的函式與樣式
- **無重複定義**：避免同名函式 / class 重複
- **不誤傷**：修改時不影響無關功能
- **先確認根因再修**：不靠猜測，定位問題本質
- 每次改動後驗證：JS 語法（`node --check`）、CSS 括號配對、版本一致性

## 版本與更新機制

採用 Service Worker 快取，版本以 `APP_VERSION` 控管。

- 新版部署後，Service Worker 偵測到版本變化
- 應用程式顯示「🔄 發現新版本」橫幅
- 使用者點擊後接管並重新載入

> 若更新後畫面未變，通常是快取未更新——可完全關閉應用程式後重開，或清除網站資料。

## 未來規劃

### Capacitor 原生封裝
計畫以 Capacitor 將 PWA 封裝為 Android App，以接入原生能力：

- **Health Connect 整合**：自動讀取三星健康的運動時長、熱量、步數
- **直接開啟外部 App**（如三星健康）
- **OTA 線上更新**：網頁內容更新免重新安裝

封裝後的更新原則：

| 改動類型 | 是否需重新安裝 |
|----------|----------------|
| 網頁內容（HTML/CSS/JS） | 否（可用 OTA 線上更新） |
| 新增原生外掛 | 是 |
| 改 App 圖示／名稱／權限 | 是 |
| 升級 Capacitor 核心 | 是 |

### 其他規劃
- 英語單字本間隔複習（資料表已預留）
- 運動健康庫接入真實數據後的趨勢分析
